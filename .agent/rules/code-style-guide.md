## コードスタイル

- TypeScript/JavaScript コードは ESLint + Prettier の設定に従う
- セミコロンを使用する
- シングルクォートを使用する
- 末尾カンマは ES5 準拠
- 行の最大長は 100 文字

## コメントとドキュメント

- 公開される関数やクラスには JSDoc コメントを記述する
- 複雑なロジックには説明コメントを追加する
- TODO コメントには担当者または Issue 番号を含める

## ファイル構成

- 1 ファイルにつき 1 つのコンポーネントまたは機能
- ファイル名はケバブケース（例: user-profile.tsx）
- テストファイルはソースファイルと同じディレクトリに配置

## コミットメッセージ

- Conventional Commits 形式を使用
- プレフィックス: feat:, fix:, docs:, style:, refactor:, test:, chore:
